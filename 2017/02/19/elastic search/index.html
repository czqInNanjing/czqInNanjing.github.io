Elasticsearch Introduction

Elasticsearch is a open source search engine.
During the curriculum Software Architecture’s Assignment, my team try to use it as the search engine.

Since it is the first time to use distributed servers, it took me rather a long time to configure the servers.

In order not to repeat the identific mistakes, I will keep them down.

  1. About binding your service to public ip

    If you use a cloud provider like AWS, your serevr probably uses NAT(Network Address Translation), which means it is impossible to bind your service to a public IP address. (Your system cannot find an upcoming global address since your server do not have a physical network card)

    Therefore, in order to solve the problem, maybe you have to use private address instead to connect your servers.
