NASM简介

NASM，全称 The Netwide Assembler,是一款基于x8086和x86_64的汇编器。它具有较强的可移植性和模块性，
并且支持多种文件格式，包括Linux和’NetBSD/FreeBSD’,
‘a.out’,’ELF’,’COFF’,微软16位的’OBJ’和’Win32’能够在多种环境下运行的汇编语言。

简单而言，就是一款语法简单，适合操作系统老师用来让我们熟悉机器底层各种指令和汇编语言的工具。

NASM环境搭建

1、Mac OSX：
终端下:
brew install nasm
如果显示“command brew not found”的话请到HomeBrew官网下载这个Mac下的安装软件神器。

2、Linux：
sudo apt-get install nasm

安装完成后在命令行下敲
nasm -v
如果看到
NASM version 2.11.08 compiled on Nov 26 2015
说明安装成功。

x86和x86_64寄存器和NASM简单语法介绍

首先简单介绍下寄存器：

  1. eax、ebx、ecx、edx
    32位寄存器，约定俗成各有各的用法，但你可以选择按你的想法来放你想放的值。它们的高十六位分别为ah,bh,ch,dh,低十六位分别为al,bl,cl,dl
  2. esp、 ebp 栈指针寄存器，一般我们不改变它的值

  3. esi、 edi

    以上为通用寄存器，除此之外还有其他的寄存器，由于太多我就不一一介绍，详见一下我分享的网盘的链接： NASM 入门教材（密码uveb）

接着介绍一些常用的指令

指令名称 operator1 operator2 备注
mov des src 移动src内容至des
movzx des src 移动src内容至des，并扩展
add des src des = src + des
sub des src des = src - des
inc des null des++
dec des null des–
push src null 将src压栈
pop des null 将栈顶元素弹出至des
resb num null 在内存中保留num个字节的空间
db num null 在内存中分配1个字节的空间并用num初始化
jmp label null 无条件跳转至label处
cmp op1 op2 执行 sub op1 op2 ,比较是否为0
je label null 若cmp结果为相等，跳转
loop label null 1、ecx– 2、若ecx！= 0 ，跳转至label处
call label null 调用label处的方法，将当前地址压栈

注：

  • jmp系还有很多其他指令如jg，jng等等，不再赘述
  • resb、resw、resd、resq、rest

  • db、dw、dd、dq、dt 意义相同，其中

  • w = word = 2bytes

  • d = double word = 4bytes
  • q = 笔者不知道是啥缩写 = 8bytes
  • t = ten words = 10bytes

##关于常见指令的几个易错点：##

mov

  • 两个操作数不能都为内存，des不能为立即数
  • 两个操作数的地址长度必须一致，否则出现段错误（Segment fault）
  • 若des 长度大于 src ， 应使用movzx

push和pop call和jmp

push 和 pop 必须是一进一出，否则易出现段错误
call 和 jmp系指令 的区别在于，call会将当前地址压栈，调用ret指令后会返回执行下一条指令，而jmp只是单纯的代码跳转
